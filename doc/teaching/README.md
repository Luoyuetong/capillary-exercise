# 教学材料索引

> capillary-exercise 项目配套教学材料 · 软件工程流程与开发习惯

本目录收录"流程驱动开发"培训的理念讲解材料。**编号即讲课顺序**，从总纲到具体动作，再到收尾拔高。

| 编号 | 标题 | 定位 | 备注 |
|------|------|------|------|
| 000 | 为什么不能上来就写代码 | 开篇总纲 | 四条理由：全局思维 / 外化审视 / 领域经验 / 多角度 |
| 010 | 自我剖析：我为什么从"为什么"出发讲课 | 元说明 | **按情况选讲** |
| 020 | Issue 与模块·为什么提前规划 Issue | 具体动作的 why | Issue vs 模块；规划倒逼全局思考 |
| 030 | 为什么建议提前写测试 | 具体动作的 why | 测试是思考工具 + 约定 |
| 040 | 收尾·提前思考的内核（引言） | 收尾拔高 | 引向 `ref/` 下的咨询随笔原文 |

## 讲课主线

```
000 立总纲：动手前先思考，为什么？
  ↓
010 说讲法：我为什么重 why 轻 how（顾问定位）   ← 可选
  ↓
020 拆 Issue：提前规划如何倒逼全局思考
030 写测试：提前定义"做对的标准"
  ↓
040 收尾：所有"提前思考"的内核——定义问题 + 选择策略
```

首尾呼应：**000 问"为什么不能上来就写代码" → 040 答"那动手前到底在想什么"**。

## 共同基调

所有材料遵循同一思路：
- **从 why 出发**，每条理由配反例/对比
- **落到真实项目**（capillary-exercise）做锚点，不空谈
- **结尾点"门槛"**：过去走流程成本高、情有可原；今天工具（尤其 AI）降低了门槛，更有理由用好方法
