# 需求文档 - 领料流程

## 一、业务背景

### 1.1 系统概述
全自动劈刀发料机用于半导体封装生产中劈刀（Capillary）的自动存取管理。本期实现**领料流程（Pickup Flow）**。

### 1.2 业务场景
生产线机台需要使用劈刀时，操作员输入工单号和机台号，系统自动：
1. 查询该工单所需的劈刀类型
2. 从库存中取出最早入库的该类型劈刀（FIFO 原则）
3. 验证条码无误后出料
4. 上报 MES 系统

---

## 二、功能需求

### 2.1 领料操作界面

**FR-01：输入工单号和机台号**
- 用户在领料界面输入：
  - 工单号（Work Order）：必填，文本，50字符以内
  - 机台号（Machine No）：必填，文本，50字符以内
- 输入完成后点击"开始领料"按钮

**FR-02：显示领料进度**
- 界面实时显示当前执行步骤，例如：
  - "MES查询中..."
  - "查找库存..."
  - "PLC取料中..."
  - "读码验证中..."
  - "上报MES..."
  - "出料中..."
- 每步完成显示 ✓，失败显示 ✗ 和原因

**FR-03：显示领料结果**
- 成功：显示领出的劈刀条码、类型、原仓位
- 失败：显示失败原因（库存不足/读码失败/MES拒绝等）

### 2.2 MES 集成

**FR-04：查询所需劈刀类型**
- 输入：工单号 + 机台号
- 输出：该工单所需的劈刀类型
- 异常：
  - 工单号不存在 → 提示"工单号无效"
  - MES 服务不可用 → 提示"MES连接失败"

**FR-05：上报领料信息**
- 输入：工单号、机台号、劈刀条码、劈刀类型
- 输出：MES 确认 OK / 拒绝
- 异常：
  - MES 拒绝（如工单已关闭）→ 提示原因，劈刀放回原位并锁定

### 2.3 库存管理

**FR-06：FIFO 查找库存**
- 根据劈刀类型，查找"状态=在库"的劈刀
- 按入库时间排序（StoredTime），取最早的一个
- 无库存 → 提示"该类型劈刀库存不足"

**FR-07：更新劈刀状态**
- 领料成功后：
  - 状态：在库(0) → 已领出(1)
  - 记录关联的工单号和机台号

### 2.4 硬件交互

**FR-08：PLC 取出劈刀**
- 根据仓位坐标（Face + X + Y），指挥 PLC 从仓位取出劈刀至读码位
- 超时或失败 → 提示"PLC取料失败"

**FR-09：扫码器读码验证**
- 扫码器读取劈刀条码
- 与库存记录的条码对比：
  - 一致 → 继续
  - 不一致 → 提示"读码不符"，劈刀放回原位并锁定

**FR-10：PLC 出料**
- 读码验证通过且 MES 确认后，PLC 将劈刀放入出料口
- 操作员从出料口取走劈刀

### 2.5 异常处理

**FR-11：读码失败的处理**
- 扫码器读码失败 → PLC 将劈刀放回原仓位
- 该仓位标记为"锁定"状态，不再参与后续 FIFO 查找
- 记录异常日志

**FR-12：MES 拒绝的处理**
- MES 返回拒绝（如工单状态异常）→ PLC 将劈刀放回原仓位
- 该仓位标记为"锁定"
- 记录异常日志

**FR-13：操作日志记录**
- 每次领料操作（成功/失败）记录到 OperationLog 表：
  - 操作类型：Pickup
  - 劈刀条码、类型
  - 工单号、机台号
  - 结果：Success / Fail
  - 失败原因
  - 时间戳

## 三、非功能需求

### 3.1 性能
- NFR-01：单次领料流程总时长 < 30秒（正常情况）
- NFR-02：MES 查询超时时间：5秒
- NFR-03：PLC 单次动作超时时间：10秒

### 3.2 可靠性
- NFR-04：硬件通信失败后，系统能给出明确提示，不卡死
- NFR-05：所有操作必须记录日志，可追溯

### 3.3 可用性
- NFR-06：操作界面简洁，一屏显示，无需滚动
- NFR-07：进度提示实时更新，操作员能看到当前执行到哪一步
- NFR-08：失败原因用通俗语言描述，不显示技术错误码

### 3.4 可维护性
- NFR-09：硬件接口（PLC / 扫码器 / MES）通过接口抽象，方便切换 Mock / 真实硬件
- NFR-10：业务逻辑封装在 Service 层，UI 层只负责交互

## 四、验收标准

### 4.1 正常流程
1. 输入有效的工单号和机台号
2. MES 返回劈刀类型
3. 系统找到库存（该类型最早入库的劈刀）
4. PLC 成功取出劈刀
5. 扫码器读码成功，条码与库存记录一致
6. MES 确认上报
7. PLC 将劈刀放入出料口
8. 数据库状态更新为"已领出"，记录日志
9. 界面显示成功提示

### 4.2 异常流程
- **无库存**：界面提示"该类型劈刀库存不足"
- **读码失败**：劈刀放回原位，仓位锁定，界面提示"读码失败，仓位已锁定"
- **MES 拒绝**：劈刀放回原位，仓位锁定，界面提示"MES拒绝：工单状态异常"
- **PLC 超时**：界面提示"PLC取料超时"，不更新数据库

### 4.3 数据完整性
- 每次领料操作（成功/失败）都有日志记录
- 劈刀状态和关联工单号准确更新
- 锁定的仓位不再参与 FIFO 查找

## 五、边界与约束

### 5.1 范围边界
- **本期实现**：领料流程
- **本期不做**：存料流程、退料流程、仓位管理界面、权限管理

### 5.2 技术约束
- 开发语言：C# .NET 9
- UI 框架：WinForms
- 数据库：SQLite (.db)
- 硬件通信：进程内 Fake 模拟硬件/MES，生产环境可替换为真实硬件

### 5.3 假设与依赖
- 假设：仓库中已有入库的劈刀数据（可通过数据库脚本预置）
- 依赖：硬件/MES 由进程内 Fake 实现（`FakePlcController`/`FakeScanner`/`FakeMesService`），App 启动即可用，无需外部程序

## 六、术语表

| 术语 | 说明 |
|------|------|
| 劈刀（Capillary） | 半导体封装中的一种工具 |
| 工单（Work Order） | 生产任务单号 |
| 机台（Machine） | 生产线上的设备编号 |
| FIFO | First In First Out，先进先出原则 |
| MES | Manufacturing Execution System，制造执行系统 |
| PLC | Programmable Logic Controller，可编程逻辑控制器 |
| 仓位 | 劈刀存储位置，由 Face（A/B/C）+ X（1-18）+ Y（1-18）标识 |
| 锁定仓位 | 读码失败或 MES 拒绝后，该仓位不再参与领料，需人工处理 |
