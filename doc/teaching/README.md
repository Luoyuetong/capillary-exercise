# 教学材料索引

> capillary-exercise 项目配套教学材料 · 软件工程流程与开发习惯

本目录收录"流程驱动开发"培训的理念讲解材料。

## 编号规则

- **000 起的连续编号**（000、001、002…）：**正课内容**，按讲课顺序紧凑排列。后续新增内容（如"为什么写设计文档"）直接续 004、005…
- **整十编号**（010、020…）：**刻意后置的特殊材料**（选讲、收尾等），永远排在正课之后。用整十预留，避免正课扩充时与之冲突。

## 讲课主线

```
000 立总纲：动手前先思考，为什么？
  ↓
001 拆 Issue：提前规划如何倒逼全局思考
002 写测试：提前定义"做对的标准"
003 测试工具：xUnit / NSubstitute 是什么，两种 Mock 的区别
004 代码规范：怎么制定、怎么落地（人机分工），AI 带来什么改变
005 什么是 CI：自动跑测试，从单机开发跃迁到工程纪律
  ↓
（010 自我剖析：我为什么重 why 轻 how —— 按情况选讲，可穿插在开场附近）
  ↓
020 收尾：所有"提前思考"的内核——定义问题 + 选择策略
```

首尾呼应：**000 问"为什么不能上来就写代码" → 020 答"那动手前到底在想什么"**。

## 共同基调

所有材料遵循同一思路：
- **从 why 出发**，每条理由配反例/对比
- **落到真实项目**（capillary-exercise）做锚点，不空谈
- **结尾点"门槛"**：过去走流程成本高、情有可原；今天工具（尤其 AI）降低了门槛，更有理由用好方法
